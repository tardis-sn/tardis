TARDIS Model Input¶

This section covers how TARDIS constructs and manages supernova models.

Overview¶

TARDIS uses a mesh-first architecture where the simulation space is defined first, and physical quantities are bound to specific locations on that mesh. This provides a clean separation between topology (mesh structure) and physics (density, temperature, etc.).

Architecture Guide¶

See TARDIS Modular Architecture for the complete architectural guide explaining the philosophical foundations and design principles of TARDIS’s modular architecture. Start there to understand the conceptual framework before diving into practical usage.

Key Concepts¶

Mesh Types¶

Radial1DMesh

Spatial mesh with radii in physical units (e.g., cm)

HomologousRadial1DMesh

Velocity-based mesh with homologous expansion (\(r = v \times t\))

Mesh Locations¶

INTERFACE

Values at cell boundaries (N+1 values for N cells)

Example: radii, interface velocities

VOLUME

Values at cell centers (N values for N cells)

Example: density, temperature, abundances
